## 1. 概要

### 1.1 プロダクトビジョン

香典帳アプリ（Kouden）は、**50代の一般ユーザー**が葬儀における香典・供物・弔電の記録を**シンプルかつ効率的**に管理できるアプリです。

**「誰から何をいただいたか」がすぐわかり、「誰に何を返したか」を簡単に記録できる** ユーザー体験を提供します。

### 1.2 ミッション

> 「葬儀というデリケートな場面において、ご遺族の負担を軽減し、感謝の気持ちを確実に届けられるようにする」

### 1.3 ターゲットユーザー

| 項目 | 内容 |
|------|------|
| **年代** | 50代 |
| **ユーザー像** | 家族の葬儀で香典帳を管理する必要がある個人 |
| **利用期間** | 葬儀前後の1〜2週間（集中利用） |
| **ITスキル** | スマートフォン・PCは使えるが、複雑な操作は避けたい |
| **共有ニーズ** | 家族（兄弟姉妹など）と情報を共有したい |

### 1.4 コアバリュー


## 3. 機能要件

### 3.1 香典帳管理

---

### 3.2 エントリー管理（統合型）

> **香典・供物・弔電を1つのエントリーで管理**

| 機能 | 説明 |
|------|------|
| エントリー追加 | 名前・金額・住所などを登録 |
| エントリー編集 | 登録情報の編集 |
| エントリー削除 | 削除（確認ダイアログ付き） |
| 郵便番号検索 | 郵便番号から住所を自動入力 |
| ソート | 名前順・金額順・登録順 |
| 検索・フィルター | 名前・続柄での検索 |
| 合計金額表示 | 香典＋供物の合計を自動計算 |

**データモデル:**
```typescript
interface KoudenEntry {
  id: string;
  koudenId: string;
  
  // 基本情報
  name: string;                    // 氏名
  organization?: string;           // 会社・団体名
  postalCode?: string;             // 郵便番号
  address?: string;                // 住所
  phoneNumber?: string;            // 電話番号
  relationship?: string;           // 続柄（親戚/友人/会社など）
  
  // 香典
  koudenAmount: number;            // 香典金額
  
  // 供物
  hasOffering: boolean;            // 供物ありフラグ
  offeringDescription?: string;    // 供物の内容（例：果物、花）
  offeringAmount?: number;         // 供物の金額（わかる場合）
  
  // 弔電
  hasTelegram: boolean;            // 弔電ありフラグ
  
  // 合計
  totalAmount: number;             // 香典 + 供物の合計金額
  
  // 返礼
  isReturnCompleted: boolean;      // 返礼完了フラグ
  returnItem?: string;             // 返礼品（例：カタログギフト3,000円）
  returnDate?: Date;               // 返礼日
  
  // メモ
  notes?: string;                  // 備考
  
  // システム
  createdAt: Date;
  updatedAt: Date;
}
```

## 4. UI/UX 設計方針

### 4.1 50代ユーザー向け設計

| 項目 | 方針 |
|------|------|
| **文字サイズ** | デフォルトで大きめ（16px以上） |
| **ボタン** | 大きく、タップしやすく（最小44px） |
| **色使い** | コントラスト高め、落ち着いた配色 |
| **ナビゲーション** | 最小限（3〜4項目）|
| **入力フォーム** | 1画面に入力項目を詰め込みすぎない |
| **エラーメッセージ** | わかりやすい日本語 |
| **ダークモード** | 対応（目の負担軽減）|

### 4.2 主要画面

1. **香典帳一覧** - 作成した香典帳の一覧
2. **エントリー一覧** - 誰からいくらいただいたかの一覧（合計表示付き）
3. **エントリー登録** - シンプルな入力フォーム
4. **未返礼一覧** - まだ返礼していない人の一覧
5. **設定** - アカウント・共有設定
