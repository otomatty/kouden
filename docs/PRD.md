## 1. 概要

### 1.1 プロダクトビジョン

香典帳アプリ（Kouden）は、葬儀や法要における香典・供物の記録管理をデジタル化し、効率的かつ正確に行えるアプリです。従来の紙ベースの香典帳管理から脱却し、複数人での共同編集、リアルタイム同期、返礼品管理までを一元化します。

### 1.2 ミッション

> 「葬儀というデリケートな場面において、ご遺族の負担を軽減し、感謝の気持ちを確実に届けられるようにする」

### 1.3 ターゲットユーザー

| ユーザー層 | 説明 |
|------------|------|
| **一般ユーザー** | 家族の葬儀で香典帳を管理する必要がある個人 |
| **葬儀社** | 顧客サービスとして香典帳管理を提供する事業者 |
| **寺院・宗教法人** | 檀家の法要管理を行う宗教施設 |
| **ギフトショップ** | 返礼品販売・配送を行う事業者 |

## 2. 技術スタック

### 2.1 アーキテクチャ概要

```
┌─────────────────────────────────────────────────────────────┐
│                    Kouden Application                        │
├─────────────────────────────────────────────────────────────┤
│  Frontend: Vite + React + TypeScript + Tailwind + shadcn/ui │
├─────────────────────────────────────────────────────────────┤
│  Native Layer: Tauri v2 (Rust)                              │
│  ├── Desktop: macOS, Windows, Linux                         │
│  └── Mobile: iOS, Android                                   │
├─────────────────────────────────────────────────────────────┤
│  Local Storage: SQLite (オフラインファースト)                  │
├─────────────────────────────────────────────────────────────┤
│  Cloud Sync: Supabase (PostgreSQL + Realtime)               │
```

## 8. データベーススキーマ概要

### 8.1 アーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│                    データベース構成                          │
├─────────────────────────────────────────────────────────────┤
│  ローカル（SQLite）           クラウド（Supabase）           │
│  ├── 高速なローカルCRUD       ├── マスターデータ            │
│  ├── オフライン動作           ├── チーム共有・同期          │
│  └── 暗号化（SQLCipher）      └── バックアップ・復元        │
└─────────────────────────────────────────────────────────────┘
```

### 8.2 同期対象テーブル（ローカル ⇄ クラウド）

| カテゴリ | テーブル |
|----------|----------|
| **香典帳** | `koudens`, `kouden_entries`, `kouden_members`, `kouden_roles`, `kouden_invitations` |
| **お供え物** | `offerings`, `offering_entries`, `offering_photos` |
| **弔電** | `telegrams` |
| **返礼品** | `return_records`, `return_items`, `return_record_items`, `return_method_types` |
| **ユーザー** | `profiles`, `user_settings`, `relationships` |

### 8.3 クラウド専用テーブル

| カテゴリ | テーブル |
|----------|----------|
| **システム** | `system_announcements`, `user_announcement_reads`, `support_tickets`, `ticket_messages` |
| **監査** | `admin_audit_logs`, `kouden_entry_audit_logs`, `kouden_entry_locks` |

### 8.4 同期用メタデータ

各同期対象テーブルには以下のカラムを追加：

```sql
-- 同期メタデータ
sync_id UUID UNIQUE NOT NULL,      -- グローバル一意ID
updated_at TIMESTAMP NOT NULL,     -- 最終更新日時
version INTEGER DEFAULT 1,         -- 楽観的ロック用バージョン
is_deleted BOOLEAN DEFAULT FALSE,  -- 論理削除フラグ
local_only BOOLEAN DEFAULT FALSE   -- ローカル専用フラグ
```
